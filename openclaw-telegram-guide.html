隨身召喚：親手將 AI 助理接到手機 Telegram

本指南將帶領你一步步透過 Telegram 官方的 BotFather 建立一個專屬於你自己的機器人（Bot），取得金鑰 Token 並安全地綁定到你的 OpenClaw 中，實現手機遠端遙控！

怎麼讀每一步

  • 對 BotFather ›深色框＝在 Telegram 對話框裡，一字不差傳送給 BotFather 的指令或文字。
  • 電腦終端機（macOS / Windows）輸入 ›要在電腦的終端機或 PowerShell 裡輸入並執行的指令。
  • 對 OpenClaw 助理說 ›在手機上，傳送給你自己 OpenClaw 機器人的訊息。
  • 設定檔內容 ›OpenClaw 設定檔（JSON）裡對應的內容片段。
  • 👀 你會看到接下來終端機或聊天畫面上，大概會出現的反應。
  • ✅ 確認怎麼判斷這一步真的成功了，再往下一步。
PartA

準備工作：安裝與註冊 Telegram

確保你的手機或電腦已就緒，能接收簡訊驗證碼。

  1. 1

    在你的手機上，開啟 App Store (iOS) 或 Google Play (Android)，搜尋並下載 Telegram Messenger。

  2. 2

    打開 App，使用你的手機門號進行註冊與登入。系統會發送簡訊驗證碼至你的手機，輸入後即可完成登入。

    💡 小提醒：部分電信業者可能會阻擋國際簡訊。若一直收不到簡訊，請確認手機沒有開啟「阻擋未知簡訊」功能，或改在連線穩定的環境重試。
PartB

向 BotFather 申請機器人與 Token

在 Telegram 中與官方機器人之父互動，創立你的 Bot。

在手機或電腦版 Telegram 中執行以下步驟：

  1. 1

    在 Telegram 上方的搜尋列輸入 @BotFather。請認明有藍色勾勾的官方認證帳號，點擊進入並按下最下方的 Start 啟動對話。

  2. 2

    發送指令開始建立新機器人：

    /newbot

    👀 BotFather 會回應：詢問你要如何命名你的機器人（Display Name，例如：小明的 OpenClaw 助理）。

  3. 3

    輸入你想要的顯示名稱（這只會顯示在對話視窗上方，可以隨時修改）：

    我的 AI 助理

    👀 BotFather 會回應：好的！現在請為你的機器人設定一個唯一的帳號名稱（Username）。注意：此帳號必須以 bot 結尾（例如：xiaoming_claw_bot）。

  4. 4

    輸入唯一的帳號名稱（Username）：

    my_openclaw_2026_bot
    帳號重複？如果回應顯示 Sorry, this username is already taken... 代表被別人註冊過了。請換一個加入數字或特定代碼的名稱重試。
  5. 5

    👀 BotFather 會回應大段成功訊息：
    Done! Congratulations on your new bot... Use this token to access the HTTP API:
    緊接著會出現一串很長的英數金鑰（Token），格式類似：
    123456789:ABCdefGhIJKlmNoPQRsTUVwxyZ

    ✅ 確認：點擊或複製這串 Token 並妥善保存。不要洩漏給其他同學，因為拿到這串 Token 就能任意控制你的機器人！

PartC

將 Token 串接到 OpenClaw 並啟動

使用終端機將金鑰寫入，並重啟背景服務。

  1. 1

    另開一個新的終端機視窗（保持原本執行 openclaw 的視窗開著不要關閉）。輸入以下指令將 Token 綁定到 OpenClaw 中（請將 <Your_Token> 換成你剛才從 BotFather 複製的那一長串金鑰）：

    openclaw channels add --channel telegram --token <Your_Token>

    👀 你會看到：終端機回應 Telegram channel config updated successfully. 或者是連線設定更新成功的訊息。

  2. 2

    在同一個終端機中，執行重啟指令來套用全新的設定：

    openclaw gateway restart

    👀 你會看到：服務重新啟動，並顯示 Gateway 重新上線的成功日誌。

  3. 3

    手機送出配對請求：打開 Telegram，搜尋你剛才為 Bot 創立的 Username（例如：my_openclaw_2026_bot）。點擊 Start，並對它傳送任意文字（例如：Hello）：

    Hello

    👀 你會看到：由於 OpenClaw 的預設安全機制（pairing 模式），機器人不會直接回覆你的問候，而是會傳送一串 6 位數的配對驗證碼（Pairing Code）（例如：ABC123），並提醒你需至終端機核准此配對請求。

  4. 4

    終端機核准配對：回到電腦的終端機，執行以下核准指令（請將 <Pairing_Code> 替換成你在手機上收到的 6 位數配對碼，例如：ABC123）：

    openclaw pairing approve telegram <Pairing_Code>

    👀 你會看到：終端機回應配對成功的確認日誌。如果有需要，你也可以輸入以下指令來查看目前所有待處理的配對請求：

    openclaw pairing list telegram
  5. 5

    測試正常交談：配對核准後，再次打開手機 Telegram，對你的 Bot 傳送問候語：

    嗨！你上線了嗎？

    👀 你會看到：此時機器人已通過驗證，會立刻正常回覆你的對話，代表你的助理正式接入你的手機！

    ✅ 確認：手機成功收到回覆，且電腦上的 openclaw 終端機也有同步印出收到 Telegram 訊息的日誌。

PartD

安全防護：防盜防毒白名單設定 (dmPolicy)

只允許你控制助理，防範其他同學惡意搶奪控制權。

在預設的 pairing（配對）模式下，雖然他人必須取得配對碼並由你在終端機核准後才能使用，但若 gateway 重啟，配對狀態有時會重置。為了解決安全與連線穩定性隱憂，我們建議將安全政策改為 allowlist（白名單）限制，只允許你設定的專屬 ID 直接連線：

  1. 1

    獲取個人的專屬數字 ID：在 Telegram 搜尋 `@userinfobot`。點擊 Start，它會立刻回傳你專屬的數字 ID（一串 9-10 位數的純數字，例如：987654321）。

    ✅ 確認：記下這串屬於你的 ID 數字。

  2. 2

    終端機指令修改（主要推薦做法）：在電腦的終端機執行以下指令，直接將安全防護政策改為白名單，並設定你的數字 ID（請將 你的數字ID 替換為你在 Step 1 取得的純數字，例如 987654321）：

    openclaw config set channels.telegram.dmPolicy "allowlist"
    openclaw config set channels.telegram.allowFrom "[你的數字ID]"

    👀 你會看到：終端機回應設定更新成功的訊息。使用指令修改可以避免手動編輯 JSON 格式錯誤的風險。

  3. 3

    手動編輯設定檔（備援做法）：若上方指令執行有問題，也可以手動開啟設定檔修改。先在終端機輸入以下指令取得設定檔的絕對路徑：

    openclaw config file

    接著使用你習慣的文字編輯器（如 VS Code、記事本）打開該路徑的檔案，找到 channels → telegram 區塊，修改安全權限如下：

    "telegram": { "enabled": true, "botToken": "你申請的Token", "dmPolicy": "allowlist", "allowFrom": ["你的數字ID"] }

    ✅ 修改後儲存設定檔並關閉編輯器。

  4. 4

    在終端機重啟服務以套用安全白名單：

    openclaw gateway restart

    👀 白名單效果測試：這時候，你可以讓旁邊的同學搜尋你的 Bot，請他對你的 Bot 傳送任何訊息，看看會不會被已讀不回。然後你自己傳送一則訊息，確認只有你可以控制它。

    ✅ 確認：其他人的指令一律被阻擋（Logs 中會顯示攔截紀錄），只有你擁有這台 Agent 的專屬指揮權！

⚠️ 安全警示：如果白名單沒有設定好，別的同學只要對你的 Bot 下達 rm -rf / 或其他惡意指令，將會直接破壞你電腦上的檔案。請務必完成 Part D 白名單設定，以保護你的系統安全！
