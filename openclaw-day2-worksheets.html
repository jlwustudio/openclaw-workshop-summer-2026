怎麼讀每一步

  • 對 OpenClaw 助理說 ›深色框＝一字不差可以直接對 OpenClaw 說的話（＜ ＞ 裡的字換成你的）。
  • 👀 你會看到Agent 接下來大概會有的反應、畫面上會出現什麼。
  • ✅ 確認怎麼判斷這一步真的成功了，再往下一步。

☁️ 今天先讀：開場檢查 + 雲端額度守則（很重要）

  • 🔌先確認助理還在：如果你的電腦昨晚被還原、或今天換了一台，昨天裝好的 OpenClaw、登入狀態、素材可能都不見了。開場先在終端機打 openclaw：能正常進入就直接往下；若出現「找不到指令／未授權」，舉手找助教，照昨天安裝教學 Part B 重跑一次（裝 Ollama → ollama signin → ollama launch openclaw），再把素材重新複製進 workspace（見下方 📁）。
  • 🧠今天六個主題一律用雲端模型（昨天體驗過的本機模型太慢，不適合連續任務）。
  • 🐢每個人的免費額度各自獨立、但都有限，而且全班擠同一條教室網路。一次只交代一個任務、等它做完再下一個，不要狂敲 Enter 連發，否則大家一起變慢或被暫時擋下。
  • 📁素材在 workspace 才讀得到：講義裡 openclaw-sample-data/... 的路徑，都以 Agent 的 workspace（~/.openclaw/workspace）為準。開始任何主題前，先確認素材在裡面——如果它說找不到檔案（電腦被還原時很常見），對 OpenClaw 助理說「把 ~/Desktop/openclaw_workshop/openclaw-sample-data 複製到你的 workspace」再繼續。
  • 🔁如果它變很慢、回「額度用盡／請稍後再試」：先等 1–2 分鐘再試；還是不行就執行 ollama launch openclaw --config 切換到你電腦本機的 Ollama 模型（如 llama3.2:1b）繼續進行實作。
  • 💬講義裡的「配對分享／想一想」是故意不連網的環節——利用這些空檔讓大家的請求錯開，不是浪費時間。

主題02

整理魔法 —— 檔案狂飆與數據分析

讓 Agent 整理檔案、批次改名、分析資料。

09:55–10:50
🧰 事前準備：① 先拿練習資料夾試，不要一開始就對「下載」「文件」這種真的資料夾動手。② 有範例檔的話，把 openclaw-sample-data/theme2-檔案整理/待整理 複製到桌面，並改名為「練習」；沒有的話照下面步驟 1 讓 Agent 自己造一個同名資料夾。（後面每一步都統一叫它「練習」資料夾，兩條路線都通用。）

● 基礎任務 A · 造練習檔 → 分類 → 批次改名

  1. 1
    在桌面建一個「練習」資料夾，裡面放：兩個 txt（a.txt、b.txt）和兩張用純色產生的圖片（cat.png、sea.png）。

    👀 它會逐一建立檔案並回報。（若已經把範例檔案夾複製並改名為「練習」，就跳過這步，直接進行步驟 2。）

    ✅ 確認：桌面「練習」資料夾裡有幾個檔案（自己造的話是 4 個；用範例檔的話會更多）。

  2. 2
    看一下桌面「練習」資料夾，把圖片和文字檔分到「images」和「docs」兩個子資料夾。動手前先告訴我你打算怎麼分。

    👀 你會看到：它先列出計畫（哪些檔算圖片、哪些算文字），你回「可以」它才動手。

    ✅ 確認：資料夾裡多了 images/ 和 docs/，檔案分對了。

  3. 3
    把「練習」資料夾裡所有檔案，檔名前面加上今天的日期（格式 YYYY-MM-DD-，用今天實際的年月日）。

    👀 它會一個個改名並回報，例如「a.txt → YYYY-MM-DD-a.txt」（YYYY-MM-DD 換成今天實際的年月日）。

    ✅ 確認：每個檔名前面都多了今天的日期。

用範例「待整理」資料夾練「誰才是最終版」（只有用範例檔複製改名的「練習」資料夾才有這些檔；自己造練習檔的話跳過這題，直接做下面 CSV 練習）

看一下「練習」資料夾裡的 報告_v1.txt、報告_最終.txt、報告_最終_真的最終.txt，比較內容，告訴我哪一份才是真正該留下的最終版，並說明你的判斷依據。

👀 這是很多人電腦裡真實會發生的亂象——檔名有「最終」不代表內容真的最新最好。練習讓 Agent 幫你判斷內容，而不是只看檔名猜。

用範例 CSV 練分析

讀取「openclaw-sample-data/theme2-檔案整理/學生社團調查.csv」，告訴我哪個社團人最多、平均滿意度最高的是哪一個，用兩句話總結。
卡住了？① 它找不到資料夾 → 把完整路徑講清楚（例如「桌面的『練習』資料夾」）。② 你怕它改錯真的檔案 → 永遠先在「練習」資料夾試，並讓它先講計畫再動手。③ 想反悔 → 跟它說「把剛剛的改名還原」。

● 進階挑戰 · 讓它寫一個可重複使用的小腳本

從「做一次」升級成「一個工具」

  1. 1
    幫我寫一個小腳本 tidy.sh，執行後會把桌面所有 .png 圖片搬到桌面的「Screenshots」資料夾。先把腳本內容給我看、解釋每一行，我說可以再執行。

    👀 它會產生腳本、逐行解釋，等你同意。

    ✅ 你看得懂每一行在做什麼，再讓它跑。

  2. 2
    很好，再幫我加一行：搬完後印出「共搬移了 N 個檔案」。

    ✅ 執行後真的印出數量。你剛剛「讀懂 → 修改 → 再測」了一支程式。

解題引導：

主題03

心靈感應 —— 教 AI 記住你，訂做專屬助理

教 Agent 記住你，變成你專屬的助理。

11:05–12:00
🧰 事前準備：想好你要它記住的「可以公開」的資訊（名字、年級、興趣、講話風格）。先不要放任何密碼、身分證、別人的私事。

● 基礎任務 A · 讀取記憶 → 修改個人資訊 → 驗證與應用

  1. 1

    先測試它是否還記得你昨天在對話中或自我介紹時透露的資訊：

    你還記得關於我的哪些事？列出來。

    👀 關鍵驗證：它應該列出你昨天自我介紹的名字、興趣、目標等。這證明「記憶」有成功跨天/跨對話留存下來。

    ✅ 確認：它列出的個人資訊符合你昨天的設定。

    🔁 如果它說「不記得你」（電腦昨晚被還原時會這樣）：別緊張，先重新自我介紹一次並請它記住，就能接著往下做——

    記住關於我的事：我叫 <名字>，就讀 <年級>，興趣是 <興趣>。以後都記得。
  2. 2

    請它修改/更新你的個人資訊，例如加入新的興趣：

    請幫我修改我的個人資訊，把興趣改為 <新的興趣>（例如：寫程式、聽音樂），並多加一項「喜歡看科幻片」。

    👀 它會回覆已記住修改後的資訊。

    ✅ 確認：它成功回應已完成修改。

  3. 3

    再次查詢以確認更新成功，並用新偏好來推薦內容：

    列出你現在記得關於我的事。並根據我的新愛好，推薦三部我可能會喜歡的科幻片。

    ✅ 確認：它列出的興趣確實更新了，且推薦的影片符合「科幻片」及更新後的興趣偏好。

  4. 4

    考驗它對今天操作（上網搜尋與瀏覽）的記憶：

    根據我們剛才的聊天紀錄，我最近上網找過或關心哪些主題？

    👀 重點觀察：它會回顧前面「主題 01（網頁搜尋）」時的紀錄，摘要出你上網搜尋過的關鍵字或內容。

    ✅ 確認：它能正確回憶起剛才搜尋過的主題或網頁重點。

    🔁 如果它想不起來（例如中途重啟過對話）：別緊張，直接對它說「我今天在主題 1 查過台灣黑熊的資料，幫我記住」，再繼續下一步即可。

🔒 停下來想 10 秒：叫它「列出記得我的哪些事」，看看內容。如果這些被別人看到你會不會不舒服？只教它記無妨外流的資訊。要它忘記某件事，就說「請忘掉我剛剛說的 XXX」。

● 進階挑戰 · 檢視記憶 + 設計助理人設

把「它記得什麼、它怎麼講話」變成你能控制的

  1. 1
    把你目前記得關於我的所有事情完整列出來，我要檢查。

    ✅ 逐條看，把不想留的挑出來。

  2. 2
    忘掉「<你不想留的那條>」。然後從現在起，用「嚴格但貼心的學長」語氣跟我說話：先鼓勵、再直接指出問題。

    ✅ 接著隨便問它一題，感受語氣真的變了。你剛剛「設計了一個 AI 人設」。
