Part0開始之前：檢查電腦準備好了沒

花 5 分鐘做一次體檢，之後就不會卡關

安裝前確認這幾件事，可以避免現場一半時間花在排錯。macOS 與 Windows 的步驟幾乎一樣，下面每個地方都會用「macOS：/ Windows：」分開標。

1

確認作業系統版本

macOS：點左上角 → 「關於這台 Mac」，確認版本在 macOS 13 Ventura 以上（14 Sonoma、15 Sequoia 更好）。

Windows：按 ⊞ Win + R，輸入 winver 按 Enter，確認是 Windows 10 22H2 以上或 Windows 11。

⚠
版本太舊（例如 Windows 10 舊版、macOS 12 以下）不是不能裝，但比較容易出問題，建議先更新系統再開始。
2

確認硬碟空間與網路

建議至少 2 GB 可用空間，並確認能連上網路（Ollama 安裝檔與模型都要從網路下載）。

👀 檢查可用空間：macOS「關於這台 Mac → 儲存空間」；Windows「檔案總管 → 本機」右鍵「內容」看剩餘空間。

3

確認你有「安裝權限」

安裝 Ollama 過程可能需要輸入電腦的登入密碼（管理員權限）。如果這台電腦是學校配發、你不是管理員帳號，安裝前請先跟資訊組或助教確認你能不能安裝軟體、或請他們先幫忙裝好 Ollama。

4

建議：另外開一個「一般使用者」帳號來做這次工作坊

如果是自己的電腦，建議另外新增一個一般使用者（非管理員）帳號，工作坊期間都用這個帳號登入操作，不要直接用平常的管理員帳號。macOS：「系統設定 → 使用者與群組 → 新增使用者」，類型選標準（Standard）。Windows：「設定 → 帳戶 → 家庭與其他使用者 → 新增帳戶」，帳戶類型選標準使用者。

🛡
為什麼這樣比較安全：OpenClaw 之後會直接讀寫檔案、下指令，用一般帳號登入可以把它的活動範圍限制在這個帳號自己的家目錄裡，不容易誤動到系統設定或別的帳號的檔案，出問題也只要砍掉這個帳號重建就好，不會弄髒（污染）或弄壞整台電腦。安裝當下需要輸入密碼的地方，直接輸入管理員帳號的密碼授權即可（macOS/Windows 都允許用一般帳號登入、臨時用管理員密碼完成單次安裝），裝好之後平常使用完全不需要用到管理員帳號。

教室共用電腦通常本來就是用一般帳號登入（管理員密碼由資訊組保管），這一步可以直接跳過。

PartA認識免費大腦：Ollama Cloud

OpenClaw 需要一顆「大腦」，我們用 Ollama Cloud

OpenClaw 本身不含 AI 模型，需要接一個 LLM。我們用 Ollama Cloud 的免費方案——大模型跑在雲端，你的電腦不用下載幾十 GB 的模型檔、也不需要高階顯示卡。這一段先看懂它怎麼運作，實際安裝與登入在 Part B。

1

它是怎麼運作的

等一下你會在自己電腦裝一個小小的 Ollama 程式，它負責當「橋樑」：把你在 OpenClaw 打的話送到 Ollama 的雲端伺服器上、由大模型運算，再把結果送回來。你只要用自己的 Ollama 帳號登入一次（Part B 會用 ollama signin 指令，開瀏覽器授權），就能免費用雲端模型，不需要手動申請、複製貼上任何金鑰。

ℹ
登入時如果還沒有 Ollama 帳號，授權頁面可以直接用 Email 或既有帳號免費註冊，不需要信用卡。建議用自己平常看得到的個人信箱，工作坊結束後用量紀錄都還找得到。
2

認識一下你的免費額度

免費方案不是無限用：額度是用「跑模型所花的運算時間」計算，不是次數。session 額度每 5 小時重置、週額度每 7 天重置，同時間只能跑一個雲端模型。這對工作坊來說完全夠用——正常對話、寫個檔案、抓個網頁都不太會碰到上限，但選特別巨大的模型會比較快把額度用掉（Part B 選模型時會再提醒）。

💡
每位同學都用自己的 Ollama 帳號（而不是共用），額度各自獨立計算，不會因為隔壁同學狂發問而害你被限速。想查自己實際用量，登入後可到 ollama.com/settings 頁面查看。
🔑
進階（選讀）：改用 API 金鑰。大部分同學用 ollama signin 登入就好。若你想用「金鑰」的方式（例如之後要寫程式串接、或在無法開瀏覽器的環境），可登入 ollama.com/settings/keys 按 Add API Key 產生一把（畫面只顯示一次，複製後貼到安全的地方），再把它設成環境變數 OLLAMA_API_KEY。金鑰＝密碼，不要外流。

PartB裝 Ollama、登入、一鍵啟動 OpenClaw

核心就一行：ollama launch openclaw

先裝好 Ollama、登入一次，再用一行指令讓 Ollama 自動把 OpenClaw 裝好並啟動。macOS 與 Windows 的差別只在「怎麼裝 Ollama」跟「終端機叫什麼名字」，下面都會分開標。

📚
ollama launch openclaw 是較新的指令，實際畫面文字可能隨版本略有不同，照著它問的問題選就好，並可對照本教學的畫面。可參考 Ollama 官方文件（docs.ollama.com/integrations/openclaw）。
1

安裝 Ollama

請依你的系統至官網下載安裝包，或直接在終端機執行指令安裝：

🍎 macOS 安裝方式

至官網下載並安裝 macOS .dmg 安裝檔（雙擊打開並拖入「應用程式」資料夾）。

或是使用終端機安裝：若尚未安裝 Homebrew，請先貼上下列指令安裝：

$ /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

* 安裝完成後，請務必將目前的終端機視窗關閉重新打開，以利載入 brew 環境變數。

接著再執行以下指令安裝 Ollama：

$ brew install --cask ollama

💻 Windows 安裝方式

至官網下載並執行 Windows .exe 安裝檔，或是直接在 PowerShell 執行：

PS> winget install Ollama.Ollama

🚨 重要提醒：安裝完成後（特別是使用終端機指令安裝者），請務必先啟動 Ollama 應用程式以開啟背景服務：

  • macOS：請至「應用程式」資料夾雙擊打開 Ollama（首次開啟會問權限，請按同意）。
  • Windows：請在開始功能表搜尋並開啟 Ollama，確認電腦右下角工作列/系統匣出現 Ollama 經典羊駝圖示。

接著打開終端機驗證（macOS 用「終端機 / Terminal」；Windows 用「Windows Terminal」或「PowerShell」）：

$ ollama --version ollama version is 0.x.x

看到版本號代表 Ollama 裝好了。若出現 command not found / not recognized，把終端機整個關掉、重新打開再試一次（新裝好的指令要開新視窗才生效）。

2

登入你的 Ollama 帳號（解鎖免費雲端模型）

在終端機輸入：

$ ollama signin

👀 你會看到：它會自動打開瀏覽器（或印出一個網址請你自己貼到瀏覽器）到 Ollama 的授權頁面。還沒有帳號的話，可以在這頁直接免費註冊（Email 或既有帳號皆可，不需信用卡）。按頁面上的同意 / 授權把這台電腦跟你的帳號綁定，終端機就會顯示登入成功。

ℹ
這一步就是 Part A 說的「登入一次」。之後這台電腦都記得你，不用每次重登。要換帳號或登出可用 ollama signout。
3

先確認電腦有 Node.js 與 Git

下一步的 ollama launch openclaw 會用 npm 幫你自動安裝 OpenClaw（npm 來自 Node.js），且執行時需要 Git。若你的電腦尚未安裝，請依系統進行安裝：

🍎 macOS 安裝方式

通常已內建 Git。若尚未安裝 Node.js，可直接至 nodejs.org 下載安裝 LTS 版本安裝包。

或是使用終端機安裝：若尚未安裝 Homebrew，請先貼上下列指令安裝：

$ /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

* 安裝完成後，請務必將目前的終端機視窗關閉重新打開，以利載入 brew 環境變數。

接著再執行以下指令安裝 Node.js 與 Git：

$ brew install node git

💻 Windows 安裝方式

請在 PowerShell 貼上這兩行安裝，然後把整個終端機關閉重新打開：

PS> winget install OpenJS.NodeJS.LTS
PS> winget install Git.Git

（若無 winget，請手動下載安裝 Node.js LTS 與 Git 安裝檔。）

不確定裝好沒？在終端機輸入 node -v 與 git --version，若有正常顯示版本號即代表成功！

4

一行指令啟動 OpenClaw

在終端機輸入這行，按 Enter：

$ ollama launch openclaw

👀 你將會經歷的互動流程：

  • 安裝詢問：會看到 OpenClaw is not installed. Install with npm?，選 Yes 按 Enter 開始下載安裝。
  • 選擇模型：跳出 Select models for OpenClaw，建議選擇雲端模型 minimax-m3:cloud（或帶有 :cloud 的其他雲端模型，如 kimi-k2.7-code:cloud），按 Enter 確認。
    * 註：若此時瀏覽器沒有登入 Ollama，系統會彈出瀏覽器並要求你登入授權。
  • 安全性確認：提示 OpenClaw can read files and run actions... I understand the risks. Continue?，選 Yes 繼續。
  • 通訊頻道連結：提示 Connect a channel (messaging app) now?，可先選 Set up later 跳過。
  • 啟動與畫面：服務啟動後，會印出 Web UI 連結（例如 http://127.0.0.1:18789/#token=...），並自動進入終端機的 TUI（終端機聊天介面）。此時畫面會顯示版本號 OpenClaw 2026.6.11 (e085fa1)，看到 Wake up, my friend! 即代表成功！
⚠
選模型的小提醒：免費額度是用運算時間算的，模型越大、回應越慢、越快把 5 小時額度用掉。挑一個中小型的雲端模型即可，日常對話、寫檔案、抓網頁綽綽有餘；避免一開始就挑最巨大的（例如 400B 以上）或帶 preview 字樣的。
⚡
想更快、跳過選單？可直接指定模型並自動同意：ollama launch openclaw --model kimi-k2.7-code:cloud --yes。只想重新設定、先不啟動：ollama launch openclaw --config。
🧩
進階（選讀）：不透過 Ollama、自己裝 OpenClaw。你也可以用 curl -fsSL https://openclaw.ai/install.sh | bash（或 npm i -g openclaw）先裝好 OpenClaw，再執行 openclaw，在設定精靈裡選 Ollama Cloud 當 provider。用 ollama launch openclaw 的好處是上面這些它都幫你做掉了，新手建議直接走一行指令那條路。

PartC第一次動手做事

叫醒你的 Agent

上一步 ollama launch openclaw 已經把你帶進 OpenClaw 聊天畫面了。這一段先認識「workspace」，再跑第一個任務確認它真的會動手做事。

1

認識「workspace」——Agent 的預設工作資料夾

OpenClaw 的 Agent 有一個自己的 workspace 工作資料夾，預設固定在 ~/.openclaw/workspace——跟你在終端機的哪個位置啟動 openclaw 無關。你講的相對路徑（例如 openclaw-sample-data/...）都會以這個資料夾為準。

為了讓 Agent 直接讀到後面課程的範例檔，請將您複製或下載下來的 openclaw_workshop 專案資料夾底下的 openclaw-sample-data 複製進 workspace。請先在終端機切換（cd）至該專案資料夾下，並依作業系統執行以下指令：

macOS（終端機）：

$ cp -R openclaw-sample-data ~/.openclaw/workspace/

Windows（PowerShell）：

PS> Copy-Item -Recurse openclaw-sample-data "$env:USERPROFILE\.openclaw\workspace\"
2

跑第一個任務驗證

對它輸入一句話，例如：

› 在桌面建立一個 hello.txt，內容寫「OpenClaw 啟動成功！」 ✓ 已建立 ~/Desktop/hello.txt

去桌面看看 —— 如果檔案真的出現了，代表你的 Agent 已經會「動手做事」了。

3

如何完全關閉與停止它（非移除）

若你想暫時釋放記憶體或完全關閉 AI 助理，請依序關閉 OpenClaw 與 Ollama：

1. 關閉 OpenClaw 聊天介面與背景服務：

  • 若是在還在執行的終端機畫面，可以連按兩次 Ctrl + C 即可退出或強制中止執行。
  • 由於 OpenClaw 有背景服務（Gateway）在運作，請在終端機輸入以下指令完全停止背景服務：
$ openclaw gateway stop
💡
Windows PowerShell 權限提示：若執行上述指令出現「因為此系統上已停用指令碼執行」錯誤，請先在 PowerShell 執行：Set-ExecutionPolicy RemoteSigned -Scope CurrentUser（輸入 Y 確認），或直接改用 npx openclaw gateway stop 執行。

2. 完全結束 Ollama：

  • macOS：在螢幕右上方選單列（Menu Bar）找到 Ollama 圖示（一隻羊駝），點擊右鍵選 Quit Ollama。
  • Windows：在工作列右下角系統匣（System Tray）找到 Ollama 圖示，點擊右鍵選 Quit。
4

下次怎麼再打開它

之後不用每次都跑一整套安裝。你可以直接在終端機輸入以下指令來開啟網頁版介面（Dashboard）：

$ openclaw dashboard
💡
背景服務未啟動？若此時背景服務尚未開啟，系統會自動詢問：
Gateway is not running. Start it now so OpenClaw can open the dashboard? [Y/n]
此時請輸入 Y（或直接按 Enter），系統便會自動幫你把背景服務啟動並開啟網頁。
💡
Windows 提示：若出現停用指令碼錯誤，請改用 npx openclaw dashboard。

如果你想使用終端機畫面 (TUI) 聊天：

請直接輸入以下指令（在哪個目錄底下打都可以，Agent 的 workspace 不會變）：

$ openclaw

若需要重新選模型或重跑設定，請執行 ollama launch openclaw --config 即可。

✓ 你完成了！確認這五件事都 OK：

  • 終端機打 ollama --version 有版本號，且 ollama signin 已登入成功。
  • 安裝完成後，進入 TUI 聊天畫面或輸入 openclaw --version 確認版本號為 OpenClaw 2026.6.11 (e085fa1)。
  • 模型是一個 :cloud 雲端模型，沒有「未登入 / 未授權」訊息。
  • 桌面出現了 hello.txt，打開內容正確。
  • 範例素材 openclaw-sample-data 已經複製到 ~/.openclaw/workspace/ 裡（Day 2 六個主題都靠它）。

PartD安裝後小抄

常用指令 & 設定放在哪裡

裝好之後不用整份教學重看一次，遇到下面這些狀況直接查這張小抄。

openclaw

啟動 Agent、進入聊天畫面。日常使用只需要這一行。

openclaw --version

查看當前安裝的 OpenClaw 版本號與提交代碼資訊。

openclaw gateway install / restart / stop / status

安裝、重新啟動、停止或查看 OpenClaw 背景服務（Gateway）的守護狀態。

openclaw logs --follow

即時追蹤與觀察背景服務的運作日誌，可以用來排除連線與 API 錯誤。

openclaw doctor

自動診斷系統狀態，稽核並修復背景服務的設定偏移。

openclaw configure --section web

設定搜尋引擎（如 Brave API Key）以啟用網頁搜尋工具。

openclaw skills

瀏覽與管理外掛技能（Skills），可用來擴充助理的實作能力。

ollama launch openclaw

當初的一鍵安裝／啟動指令。裝過之後也可以再跑一次，它會沿用設定直接開起來。

ollama launch openclaw --config

只重新設定（換模型、重選 provider）而不馬上啟動。想換一個雲端模型時用這個最快。

ollama list

列出你電腦上／可用的模型；帶 :cloud 的是雲端模型。

ollama signin / ollama signout

登入 / 登出 Ollama 帳號。換帳號、或這台電腦要交給別人時用得到。

npm install -g openclaw@2026.6.11

手動安裝或指定更新至本教學指定的特定版本 2026.6.11。

📁
設定存在哪？安裝時終端機的訊息會顯示實際路徑（通常在你的使用者家目錄下、一個以 OpenClaw 命名的隱藏資料夾）。想重新設定模型，最簡單就是跑 ollama launch openclaw --config，跟著畫面走一次；每個版本引導畫面用字可能不同，照畫面問的做即可。
🔁
教室共用電腦、下一堂課要重來一次？完整的解除安裝＋恢復乾淨步驟寫在下面 Part E。

PartE解除安裝 OpenClaw 與 Ollama

工作坊結束了？把電腦恢復原狀

不管是活動結束後不想繼續留著，還是教室共用電腦要交給下一堂課用，照著下面步驟就能乾淨移除，不會留下殘留檔案。

1

移除 OpenClaw 主程式

OpenClaw 是 ollama launch 用 npm 幫你裝的，一行指令移除：

$ npm uninstall -g openclaw

若當初是自己用 curl 一鍵安裝的（進階選讀那條路），先找出執行檔位置再刪：

$ which openclaw /usr/local/bin/openclaw $ rm "$(which openclaw)"
2

移除設定資料夾

光移除主程式還不夠——設定檔還留在使用者家目錄下的一個資料夾裡。如果安裝時終端機曾顯示過確切路徑，直接用那個路徑最準；不確定的話，用下面指令先找出來看看，確認真的是 OpenClaw 的資料夾後再刪除：

macOS：

$ find ~ -maxdepth 2 -iname "*openclaw*" /Users/yourname/.openclaw $ rm -rf ~/.openclaw

Windows：打開「檔案總管」，在網址列貼上 %APPDATA% 按 Enter，找到 openclaw 資料夾，右鍵刪除。

3

登出並移除 Ollama

先從這台電腦登出你的 Ollama 帳號，避免下一個人用到你的免費額度：

$ ollama signout

接著，將 Ollama 軟體從系統中完整移除：

  • macOS：請先在右上角選單列（Menu Bar）的 Ollama 圖示上點右鍵選擇 Quit Ollama 結束程式，再到「應用程式」資料夾將 Ollama 拖進垃圾桶移除。
  • Windows：請先在右下角工作列系統匣的 Ollama 圖示上點右鍵選擇 Quit 結束程式，再到「設定 → 應用程式 → 安裝的應用程式」中搜尋 Ollama 並點選「解除安裝」。
4

驗證真的移除乾淨了

打開一個新的終端機視窗（舊視窗可能還記得舊指令），輸入以下指令確認皆已移除：

$ openclaw --version zsh: command not found: openclaw $ ollama --version zsh: command not found: ollama

看到「command not found」（Windows 會顯示「不是內部或外部命令」之類的錯誤），就代表兩者都已移除乾淨。

🔑
帳號還在雲端：解除安裝不會刪除你的 Ollama 帳號——那是存在 Ollama 那邊的東西。ollama signout 只是讓這台電腦忘記你。若你當初有另外產生 API 金鑰（進階選讀），不用了可到 ollama.com/settings/keys 把它刪掉。

✓ 恢復乾淨了嗎？確認這幾件事：

  • 終端機打 openclaw --version 顯示「找不到指令」。
  • 使用者家目錄下的 OpenClaw 設定資料夾已經不見了。
  • 已執行 ollama signout 且已移除 Ollama 軟體（終端機打 ollama --version 顯示「找不到指令」）。

FAQ?卡住了？常見狀況與解法

先看這裡，再舉手

command not found: ollama

Ollama 沒裝好，或裝完視窗沒重開。關掉終端機、重新打開再試一次；還是不行就回 Part B 步驟 1 重新下載安裝一次。

ollama launch：unknown command / 沒有這個指令

launch 是較新版本才有的指令。回 ollama.com/download 下載最新版覆蓋安裝，重開終端機再試。

command not found: openclaw

通常是 ollama launch openclaw 還沒跑完、或視窗沒重開。關掉終端機、重新打開，再跑一次 ollama launch openclaw。

出現「Hi, I'm Crestodian」且無法執行一般指令 / 顯示 Gateway not reachable

這代表 OpenClaw 的背景服務（Gateway）沒有正常啟動或連線中斷。請在終端機執行 openclaw gateway restart（若 Windows 遇到執行權限阻擋，請改用 npx openclaw gateway restart）來重新啟動背景服務；若依然不行，可執行 openclaw doctor 診斷系統狀態，或用 openclaw logs --follow 查看錯誤日誌。

網頁 Web UI 畫面出現「需要驗證」（要求輸入權杖 Token 或密碼）

這是因為開網頁時網址沒有帶上 Token。請在終端機執行 openclaw dashboard（或 Windows 用 npx openclaw dashboard），這會自動開啟帶有正確權杖 Token 的瀏覽器分頁。或者執行 openclaw dashboard --no-open 取得 Token 後自行複製貼上。

launch 時說找不到 npm / node

ollama launch 要用 npm 裝 OpenClaw。Windows 回 Part B 步驟 3 裝 Node.js 並重開視窗；macOS 執行 brew install node 或到 nodejs.org 下載安裝。

launch 時出現 Error: OpenClaw is not installed and required dependencies are missing (git)

這代表系統中缺少 Git。Windows 請用 PowerShell 執行 winget install Git.Git 安裝（或到 git-scm.com 下載），安裝後一定要重開終端機再試一次。

未登入 / 401 / unauthorized（用不了雲端模型）

還沒登入 Ollama 帳號。執行 ollama signin，在跳出的瀏覽器頁面按同意 / 授權，回終端機看到登入成功再重跑 ollama launch openclaw。

permission denied（權限不足）

安裝 Ollama 需要電腦權限。依提示輸入你的電腦登入密碼（打字時看不到是正常的），按 Enter。學校電腦不是管理員的話，請助教／資訊組幫忙。

下載很慢 / 卡住不動

先確認有連上教室網路（Ollama 安裝檔、模型、OpenClaw 都要下載）。等 5 分鐘仍無反應，按 Ctrl + C 中斷，重跑一次。

額度用完 / rate limit（Ollama Cloud）

Ollama Cloud 免費額度以運算時間計算，session 每 5 小時、週額度每 7 天重置。稍等重置再試，或用 ollama launch openclaw --config 換一個較小的模型。

PowerShell：「因為在此系統上已停用指令碼執行」

這是 Windows 預設的安全政策擋住腳本執行。不需要管理員權限，直接在現有的 PowerShell 視窗輸入 Set-ExecutionPolicy RemoteSigned -Scope CurrentUser，按 Enter 後輸入 Y 確認，就可以順利執行 npm 或 openclaw 指令了。

macOS：「無法打開，因為它來自身分不明的開發者」

這是 Gatekeeper 安全檢查，安裝 Ollama app 時可能出現。到「系統設定 → 隱私權與安全性」，捲到底部會看到一則允許打開此 App 的提示，點仍要打開即可，只需做一次。

Windows：SmartScreen「Windows 已保護您的電腦」

下載安裝檔（Ollama、Node.js）第一次執行常見。點畫面上其他資訊，再點仍要執行即可繼續安裝。

學校網路 / VPN 擋住連線

若下載或安裝一直卡在同個地方，可能是校網防火牆擋了特定網域。回頭看 Part 0 步驟 2 列出的網域清單，請資訊組確認是否開通；也可以先換成手機熱點測試看看是不是網路問題。

硬碟空間不足（no space left on device）

清一下下載資料夾、資源回收桶／垃圾桶再重試。雲端模型不佔你的硬碟，主要是 Ollama app 與 OpenClaw 本身需要一點空間。

🔒
最後提醒：你的 Ollama 帳號登入狀態只存在自己的電腦裡，不要把帳號密碼或 API 金鑰外流、共用。教室共用電腦用完後，記得執行 ollama signout 登出（詳見 Part E）。
